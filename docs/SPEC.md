# Техническое задание: Система управления складом (Warehouse System)

# **1\. Общее описание**

Данный проект представляет собой микросервисное приложение для учета товарных запасов на складе. Приложение позволяет отслеживать наличие товаров, регистрировать их поступление и списание, а также управлять каталогом. Проект хорошо подходит для изучения транзакционности в Spring Data JPA и ролевой модели авторизации.

# **2\. Ролевая модель**

**В системе предусмотрено две основные роли:**  
**• Administrator (Админ):** Может добавлять новые товары в каталог, редактировать их описание, а также оформлять поступления и списания (изменять остатки).  
**• User (Пользователь/Менеджер):** Может просматривать каталог товаров, проверять текущие остатки и просматривать историю движения товаров (без права изменения).

# **3\. Основной функционал**

**1\. Управление каталогом товаров (Items):**  
   \- Просмотр списка товаров (с фильтрацией по категориям и поиском по названию).  
   \- Создание нового товара с указанием артикула, названия, категории и минимально допустимого остатка.

**2\. Учет остатков (Inventory / Movements):**  
   \- Регистрация поступления товара (увеличение количества).  
   \- Регистрация списания/отгрузки товара (уменьшение количества). Валидация: нельзя списать больше, чем есть на складе.  
   \- Просмотр текущего остатка по конкретному товару.  
   \- История движения (лог всех операций прихода и расхода с указанием даты и пользователя).

# **4\. Архитектура и стек технологий**

**• Backend:** Java 17+, Spring Boot (Spring Web, Spring Data JPA, Spring Security для ролевой модели).  
**• База данных:** PostgreSQL (таблицы: пользователи, товары, остатки, история движения).  
**• Кэширование:** Redis (кэширование часто запрашиваемых категорий и карточек товаров).  
**• Брокер сообщений:** Kafka (асинхронная отправка события \`LowStockAlertEvent\`, если после списания остаток товара падает ниже минимально допустимого уровня).  
**• Инфраструктура:** Docker, Docker Compose.

# **5\. Спецификация API**

Спецификация REST API в формате OpenAPI 3.0 (OAS3) вынесена в отдельный приложенный файл warehouse\_openapi.yaml.
